# 项目上下文：峨眉派工业物联网电机故障检测系统

## 技术栈
- **采集端**：ESP32-C3, ESP-IDF 框架。传感器使用 ADXL345 (SPI接口) 和 DHT11。
- **边缘端**：CV1812H 华山派开发板，运行 Linux，使用 C/C++ 开发，TPU 算力 0.5 TOPS。
- **通信**：Wi-Fi + MQTT 协议。
- **显示**：LVGL + 轻量级 Web 服务 (双模式)。
- **核心算法**：基于 1D-CNN 的故障诊断模型，部署为 cvimodel 格式。

## 架构与规范
- **职责分离**：ESP32-C3 负责采集和声光报警；华山派负责核心 AI 推理、数据显示。
- **故障等级定义**：严格遵守标准：
    - Level 0: 正常 (绿灯)
    - Level 1: 警告 (黄灯)
    - Level 2: 危险 (红灯, 蜂鸣器间歇)
    - Level 3: 严重 (红灯闪烁, 蜂鸣器持续)
- **数据流规范**：
    - 数据格式统一为 JSON：`{"accel_x": ..., "accel_y": ..., "accel_z": ..., "temp": ..., "humi": ...}`
    - 华山派通过 MQTT 回传推理结果的指令格式：`{"fault_level": 0/1/2/3}`。

## 安全与测试
- **禁止行为**：不得在代码中硬编码 Wi-Fi 密码或 API 密钥；所有输入必须校验。
- **测试要求**：在提交任何代码前，应确保已添加或更新基本的单元测试。

## 代码风格
- **ESP32-C3**：遵循 ESP-IDF 编码指南。
- **华山派**：遵循 Linux 内核代码风格。
