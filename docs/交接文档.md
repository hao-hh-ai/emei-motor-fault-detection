## 一、项目概述

工业物联网电机故障检测系统，三层架构：ESP32-C3 振动采集 → PC 桥接中枢 → 华山派 CV1812H 边缘 AI。

**核心功能**：ADXL345 三轴 50Hz 采集 → MQTT 上报 → 滑窗(256点) → 8维时域特征(AC耦合) → AI/规则双模判定 → LED+蜂鸣器反馈。

## 六、AI 模型

### 6.1 模型架构

```
Conv1D MLP (TPU, 325 params):
  features(1,8,1) → Conv1d(8→16,k=1) → ReLU
                  → Conv1d(16→8,k=1) → ReLU
                  → Conv1d(8→4,k=1) → argmax → 0/1/2/3

纯C MLP (CPU, 316 params, TPU 兜底):
  features(1,8) → Linear(8→16) → ReLU
                → Linear(16→8) → ReLU
                → Linear(8→4) → argmax
```

### 6.2 训练数据

- 来源：bridge.py 运行时自动采集规则系统判定结果
- 文件：`huashan/bridge/training_data.csv`
- 样本量：9380 条 (L0:2097, L1:3707, L2:3265, L3:311)
- 每 20 条打印实时统计

### 6.3 训练命令

```powershell
cd models/training
py -3.13 train_deploy_mlp.py
```

### 6.4 ONNX→cvimodel 转换

```bash
# 关键: Linear(Gemm) → Conv1d(kernel=1) 替代, 避免 TPU-MLIR Gemm 崩溃
# 在 VM Docker 中运行:
docker run --rm -v $(pwd):/workspace -w /workspace sophgo/tpuc_dev:latest bash -c '
  pip install tpu_mlir -q
  model_transform --model_name mc --model_def deploy_conv.onnx --input_shapes [[1,8,1]] --mlir mc.mlir
  model_deploy --mlir mc.mlir --quantize BF16 --processor cv181x --model deploy_conv.cvimodel
'
```

## 七、时域特征 & 阈值

### 8维特征（AC耦合后计算）
| 特征 | 物理意义 | 单位 |
|------|---------|------|
| RMS | 振动总能量 | m/s² |
| Peak | 最大冲击 | m/s² |
| Crest Factor | 早期冲击预警 | 无量纲 |
| Kurtosis | 信号尖峰程度 (>3故障) | 无量纲 |
| Skewness | 信号不对称性 | 无量纲 |
| Clearance Factor | 间隙变化 | 无量纲 |
| Shape Factor | 波形变化 | 无量纲 |
| Impulse Factor | 脉冲冲击 | 无量纲 |

### 阈值来源
基于 CWRU 轴承数据集统计分析，配置在：
- PC: `huashan/bridge/threshold_config.py`
- 华山派: `huashan/src/threshold_config.h`

### 加权评分
7条规则各有权重(RMS×2, Kurtosis×2, Peak×1.5, CF×1.5, Clearance×1.5, Impulse×1, Shape×0.5)，加权平均后三级阈值：0.8(WARN) / 1.8(DANGER) / 2.5(CRITICAL)，配合3窗口持续性滤波防误报。

```powershell

# 2. 启动 Bridge (数据采集自动开启 + TCP 转发)
py -u huashan/bridge/bridge.py --broker localhost --port 4883 --huashan 192.168.150.2:9999
```

### 华山派 (交叉编译 → 部署)

**编译 (VM 192.168.88.128, xiaohuihui/lihao520.):**

**部署到 华山派 (192.168.150.2, root/cvitek):**
```bash

cd huashan && py -3.13 -m http.server 8890

# 华山派上下载
cd /mnt/data
wget http://192.168.150.3:8890/huashan_bridge -O huashan_bridge
wget http://192.168.150.3:8890/deploy_conv.cvimodel -O deploy_conv.cvimodel
```

**运行:**
```bash
killall huashan_bridge

```

输出标签说明：
- `[TPU]` — Sophon TPU 推理 (需要 deploy_conv.cvimodel)
- `[MLP]` — 纯C MLP 推理 (TPU 不可用时自动兜底)
- `[rule]` — 规则系统 (MLP 也不可用时最终兜底)

## 十二、已知问题

1. **DHT11 偶尔超时** — 单总线协议受 FreeRTOS 任务抢占干扰。
2. **华山派根分区已满(100%)** — 部署必须用 `/mnt/data/` (2.3G剩余)。
3. **华山派 UTF-8 乱码** — 终端编码非 UTF-8, box-drawing 字符显示为乱码，不影响功能。
4. **Mosquitto 自动重启** — Windows 服务版 Mosquitto 可能与手动启动冲突。
5. **LED 延迟 4-5秒** — 256点滑窗固有限制，快车道方案可缓解（待合并到主线）。
6. **CI build 待确认** — 4/5 通过，ESP-IDF 编译需要 `sdkconfig.defaults` + `Kconfig.projbuild`。
