## 一、项目概述

工业物联网电机故障检测系统，三层架构：ESP32-C3 振动采集 → PC 桥接中枢 → 华山派 CV1812H 边缘 AI。

**核心功能**：ADXL345 三轴 50Hz 采集 → MQTT 上报 → 滑窗(256点) → 8维时域特征(AC耦合) → 加权评分 → 四级故障判定(0~3) → LED+蜂鸣器反馈。

## 六、时域特征 & 阈值

### 8维特征（AC耦合后计算）
| 特征 | 物理意义 | 单位 |
|------|---------|------|
| RMS | 振动总能量 | m/s² |
| Peak | 最大冲击 | m/s² |
| Crest Factor | 早期冲击预警 | 无量纲 |
| Kurtosis | 信号尖峰程度 (>3故障) | 无量纲 |
| Skewness | 信号不对称性 | 无量纲 |
| Clearance Factor | 间隙变化 | 无量纲 |
| Shape Factor | 波形变化 | 无量纲 |
| Impulse Factor | 脉冲冲击 | 无量纲 |

### 阈值来源
基于 CWRU 轴承数据集统计分析，配置在：
- PC: `huashan/bridge/threshold_config.py`
- 华山派: `huashan/src/threshold_config.h`

### 加权评分
7条规则各有权重(RMS×2, Kurtosis×2, Peak×1.5, CF×1.5, Clearance×1.5, Impulse×1, Shape×0.5)，加权平均后三级阈值：0.8(WARN) / 1.8(DANGER) / 2.5(CRITICAL)，配合3窗口持续性滤波防误报。

## 八、编译 & 启动

### ESP32-C3 (Windows, VS Code ESP-IDF 扩展)
```
Build → Flash (UART COM11) → Monitor
```

### PC Bridge (Windows, 需要 Mosquitto 先启动)
```powershell
# 1. 启动 Mosquitto
D:\mosquitto\mosquitto.exe -v -c D:\ESP32-C3\equipmenttest_project\mosquitto.conf

# 2. 启动 Bridge (含华山派 TCP 转发)
py -u huashan/bridge/bridge.py --huashan 192.168.150.2:9999
```

### 华山派 (交叉编译 → 部署)
```bash
# 虚拟机编译
cd ~/桌面/huashan
export CROSS_COMPILE=/home/xiaohuihui/huashan/workplace/sophpi-huashan/host-tools/gcc/riscv64-linux-musl-x86_64/bin/riscv64-unknown-linux-musl-
make clean && make -j4

# 上传 & 运行
scp -O huashan_bridge root@192.168.150.2:/mnt/data/
ssh root@192.168.150.2 './mnt/data/huashan_bridge 9999'
```

### 调试 (ESP32-C3 USB JTAG)
```powershell
# 一键启动
双击 firmware/gdb_debug.bat

# 手动
openocd -f firmware/openocd.cfg
riscv32-esp-elf-gdb build/emei_fault_detection.elf -ex "target extended-remote :3333"
```

### 质量门禁
```bash
bash .claude/scripts/check-firmware.sh    # 本地5道检查
# GitHub Actions 自动在 push 时触发
```

## 十一、已知问题

1. **DHT11 偶尔超时** — 单总线协议受 FreeRTOS 任务抢占干扰，偶发 "Data bit 0 timeout" 但不影响主流程。

2. **华山派根分区已满(100%)** — 部署必须用 `/mnt/data/` (2.3G剩余)。

3. **华山派 UTF-8 乱码** — 终端编码非 UTF-8，box-drawing 字符显示为乱码，不影响功能。

4. **Mosquitto 自动重启** — Windows 服务版 Mosquitto 可能与手动启动冲突，端口 1883 被占，改用 4883。

5. **Python bridge 输出缓冲** — 务必用 `py -u` 启动查看实时输出。
