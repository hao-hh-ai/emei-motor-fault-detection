# 峨眉派电机故障检测系统 — 开发交接文档

> 日期：2026-06-17  
> 当前版本：ESP-IDF v5.4.4 / ESP32-C3

## 一、项目概述

工业物联网电机故障检测系统，通过振动分析实现故障诊断。当前完成了 **ESP32-C3 传感器采集端** 的基础驱动。

## 二、硬件引脚

| 外设 | 接口 | 引脚 |
|------|------|------|
| ADXL345 | I2C | SDA=GPIO6, SCL=GPIO10, 地址 0x53 |
| DHT11 | 单总线 | GPIO18 |
| LED 绿 | GPIO | GPIO1 |
| LED 黄 | GPIO | GPIO3 |
| LED 红 | GPIO | GPIO4 |
| 蜂鸣器 | GPIO | GPIO0 |
| 串口 | UART | COM10, 115200bps |

## 五、编译 & 烧录

**方式一：VSCode 一键操作（推荐）**
1. 打开项目文件夹
2. 底部工具栏点 **🔨 Build** 编译
3. 点 **⚡ Flash** 烧录（COM10）
4. 点 **📺 Monitor** 查看串口输出（115200）

**方式二：命令行**
```bash
# 先进入 ESP-IDF 环境 (CMD)
D:\Espressif\frameworks\esp-idf-v5.4.4\export.bat
cd D:\ESP32-C3\equipmenttest_project
idf.py build          # 编译
idf.py -p COM10 flash  # 烧录
idf.py -p COM10 monitor # 监视
```

## 七、已知问题

1. **DHT11 偶尔超时** — 单总线协议对时序敏感，偶尔读取失败属正常现象。当前 1 秒轮询一次，偶尔跳过一两次不影响使用。

2. **portmacro.h IntelliSense 报错** — ESP-IDF 框架的 FreeRTOS 头文件有两行 GCC 扩展语法，VSCode IntelliSense 无法解析，但不影响编译。如影响烧录，已修改为兼容写法。

3. **COM10 被占用** — 如果烧录时报 `PermissionError`，需要先关闭串口监视器或其他占用 COM10 的程序。
