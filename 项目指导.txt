三、完整数据流
================================================================================

ADXL345(三轴振动)[I2C接口]
    └→ I2C总线
DHT11(温湿度)[单总线]
    └→ ESP32-C3(数据采集固件/ESP-IDF)
          │
          ├→ 本地GPIO控制：LED(红/黄/绿) + 蜂鸣器(声光报警)
          │
          ├→ WiFi/MQTT ─→ 华山派开发板(CV1812H/Linux SDK)
          │                 │
          │                 ├→ CPU预处理(归一化、滑窗)
          │                 ├→ TPU推理(加载cvimodel)
          │                 ├→ 输出故障等级(0正常/1警告/2危险/3严重)
          │                 ├→ 故障等级回传ESP32-C3(触发声光报警)
          │                 └→ HDMI → 4寸显示屏
          │                               │
          │                               ├→ LVGL原生界面(直接渲染)
          │                               │   ├→ 实时三轴波形图
          │                               │   ├→ 当前温湿度
          │                               │   ├→ 故障等级状态
          │                               │   └→ 声光报警状态指示
          │                               │
          │                               └→ 轻量级Web服务(浏览器访问)
          │                                   ├→ 实时三轴波形图
          │                                   ├→ 当前温湿度
          │                                   ├→ 故障等级状态
          │                                   └→ 历史数据查询
          │
          └→ 故障等级 → LED亮灯 + 蜂鸣器鸣叫


四、通信协议详情
================================================================================

传感器接口：    I2C(ADXL345) - SCL、SCK
无线协议：      WiFi 802.11 b/g/n，Station模式连接本地路由器
应用层协议：    MQTT over TCP，QoS=1
数据格式：      JSON，UTF-8编码
数据频率：      200~500Hz(可配置)
回传指令：      {"fault_level": 0/1/2/3}


五、显示方案详情
================================================================================

模式一 - 本地显示(直连HDMI)：
    技术栈：    LVGL(Light and Versatile Graphics Library)
    适用场景：  现场操作人员查看
    特点：      轻量、流畅、触控优化

模式二 - 远程访问(浏览器)：
    技术栈：    轻量级Web服务 + WebSocket
    适用场景：  远程监控、多终端访问
    特点：      可在手机/电脑上查看


六、故障等级定义
================================================================================

┌──────────┬────────────┬────────────┬──────────────┬────────────────────────┐
│ 等级     │ 状态       │ LED显示    │ 蜂鸣器       │ 说明                   │
├──────────┼────────────┼────────────┼──────────────┼────────────────────────┤
│ 0        │ 正常       │ 绿灯常亮   │ 静音         │ 设备运行正常           │
│ 1        │ 警告       │ 黄灯常亮   │ 静音         │ 振动轻微异常，建议观察 │
│ 2        │ 危险       │ 红灯常亮   │ 间歇鸣叫     │ 振动明显异常，建议检修 │
│ 3        │ 严重       │ 红灯闪烁   │ 持续鸣叫     │ 严重故障，立即停机     │
└──────────┴────────────┴────────────┴──────────────┴────────────────────────┘


七、技术栈总览
================================================================================

传感器接口：    I2C(ADXL345)、单总线(DHT11)
无线通信：      WiFi 802.11 b/g/n + MQTT协议
AI框架(训练)：  PyTorch / TensorFlow
数据集：        西储大学(CWRU)轴承数据集
模型类型：      1D-CNN(轻量级，适于边缘部署)
模型量化：      INT8量化(TPU-MLIR工具链)
模型格式：      cvimodel(CV1812H平台)
推理引擎：      算能TPU Runtime库
显示框架：      LVGL + 轻量级Web服务(双模式)
开发环境：      ESP-IDF(ESP32-C3) + Linux SDK(华山派)


八、系统特色
================================================================================

1. SPI高速采集：ADXL345通过I2C接口实现最高3200Hz采样率
2. 边缘AI推理：故障诊断在本地完成，无需上云，响应毫秒级
3. 双模式显示：LVGL直连HDMI + Web服务远程多端访问
4. 声光联动报警：三色LED + 蜂鸣器提供直观本地报警反馈
5. 端边协同：ESP32-C3采集报警，华山派推理显示，分工明确
6. 统一开发环境：ESP-IDF + Linux SDK，专业嵌入式开发工具链
